## 1. LAYER5 — `geo_exposure` (1번 아이디어)

**문제**: LAYER4가 19섹터 phase 부여 시 같은 섹터 안 종목들이 지정학 노출 천차만별인데 동일 가중 받음. 예: 반도체 SK하이닉스(HBM 중국 30%·미 수출통제 직격) vs 삼성전자(다각화) vs DB하이텍(파운드리 내수).

**해결**: 종목별 6 dim vector + 4 룰로 LAYER4 phase score에 ±조정.

### vector

- `china_revenue_pct` (0~1)
- `us_revenue_pct` (0~1)
- `taiwan_supply_dep` (0~1)
- `sanctions_risk` (0~1)
- `fx_kr_usd_beta` (-1~+1, 수출주 양수)
- `notes` (rationale 시드)

### 4 조정 룰

- 미중 strategic × china_revenue → 악재
- taiwan_strait × taiwan_supply → 악재
- us_china_tech × sanctions_risk → 악재
- hotspot_avg × fx_kr_usd_beta → 호재 (수출주, 안전자산 USD 선호)
- 합산 clamp ±0.6

### 데이터 source

- 수동 시드 — 사업보고서·IR 기반 (현 watchlist 7종)
- DART 자동 추출 — `dart_revenue_parser.py` (사업보고서 "지역별 매출" 발췌 + LLM 표준 6 region 추출, 90일 캐시)
- 한계: 회계 송장 지역 ≠ 실 end-customer 지역 (예: SK하이닉스 HBM 엔비디아 미국 인보이스로 회계 us=69%, 실 사용처는 글로벌 데이터센터). **수동 시드 override 우선 원칙**.

### 검증

5/12 12:39 SK하이닉스 single 분석에서 LLM rationale 직접 인용:

> "geo_exposure adjust -0.6 (클램프 한계) ... 미중 strategic 4.3 × 중국매출 30% → -0.16 ... 대만해협 4.5 × 대만공급 55% → -0.41 ... 기술제재 4.2 × 제재노출 65% → -0.31"

같은 섹터 반도체 내 분산:
- 000660 SK하이닉스 adjust=-0.6 (clamp)
- 005930 삼성전자 adjust=-0.336

## 2. LAYER5 — `neighbor_adjust` (2번 아이디어, 신경망)

**문제**: 종목 간 영향이 단일 신호. 공급망·계열사·그룹 자매사·경쟁사·해외 빅테크 reach가 모델 외.

**해결**: 6 edge type × 가중치 + 2-hop BFS + decay 0.5 + 한미 cross-border.

### 6 edge type

| Type | Weight | 의미 |
|---|---|---|
| supplier | +0.15 | 상류 호재 → 부품 안정 |
| customer | +0.30 | 하류 호재 → 수요 증가 |
| competitor | -0.10 | 셰어 위협 |
| subsidiary | +0.40 | 자회사·모회사 지분 직격 |
| group_peer | +0.25 | 그룹 자매사 동조 (두산밥캣↔두산에너빌리티) |
| partner | +0.20 | MOU·공동개발·전략 제휴·JV |

### 시드 source

- 수동 시드 — watchlist 7종 ego-network (엠로↔삼성SDS 모회사 0.8, 두산 그룹 3 자매사, 삼성 그룹 5종, SK 그룹 등)
- `dart_subsidiary_parser.py` — 사업보고서 "종속기업의 현황" LLM 추출 → 상장 자회사만 ticker 매핑
- `dart_partnership_parser.py` — "주요 매출처·매입처·협력사·MOU·공급계약" 발췌 → customer/supplier/partner 분류
- `auto_seed_relations_from_dart()` — 캐시 머지 후 RELATIONS 메모리 박음 (한 세션 1회 lazy)

### 누적 결과 (5/12 기준)

- 수동 시드 28 edges + DART 자동 머지 68 edges = **96 edges**
- 상장 종속 자동 추출 사례: SK(6 자회사: SK이노 52.09%·SK아이이테크 53.35%·SK텔레콤·SK스퀘어·SK네트웍스·SKC) / 한화(7) / CJ(1) / LG화학→LGES 79%
- 협력사 자동 추출: 엠로 29 edges(삼성·LG·SK·현대·POSCO·우리금융·하이브 등 광범위), SK하이닉스 4 edges(엠씨넥스 Camera Module 등), 두산 1 edge(한국가스공사 4,458억)

### 한미 cross-border (17 edges)

| 한국 종목 | 미국 이웃 | edge | 비고 |
|---|---|---|---|
| SK하이닉스 | NVDA | customer 0.9 | HBM3E 12단 단독 공급 |
| SK하이닉스 | AVGO / AMD | customer | AI 가속기 HBM |
| SK하이닉스 | MU | competitor 0.7 | D램 글로벌 3대 |
| SK하이닉스 | TSM | partner | HBM 후공정 CoWoS |
| 삼성전자 | NVDA / AAPL / QCOM | customer | 메모리·디스플레이 |
| 삼성전자 | MU / TSM | competitor | D램·파운드리 |
| SK이노 | F | customer 0.6 | Ford Blue Oval SK JV |
| 두산에너빌리티 | NNE | partner | SMR 시장 동조 |

`_load_recent_signals` 가 LAYER3/forecasts/single_us_*.json 자동 reuse — 미국 분석 결과가 한국 종목 neighbor 신호로 자동 전파.

### 2-hop BFS + decay 0.5

직접 이웃(1h)뿐 아니라 이웃의 이웃(2h, decay 0.5)도 합산. back-edge(자기 자신) 회피.

검증:
- SK하이닉스 1h만 +0.24 → 2h 합쳐 **+0.378**
- SK이노 1h 부재(이웃 신호 0건) → 2h SK하이닉스 경유 한미반도체 chain으로 **+0.018** (직접 이웃 없이도 신호 잡음)
- 두산에너빌리티 1h +0.18 → 한전기술→한국전력(subsidiary 65%) chain으로 **+0.32**
- 엠로 +0.5 clamp (1h 10건 + 2h 6건, 한국 산업 hub 입증)

### 자동화 launchd

- `com.layer3.us-cross-watchlist` — 화~토 21:00 KST (미 장 개장 전) NVDA·MU·AVGO·TSM·AMD sequential 분석. neighbor_adjust 풍부화용.
- 첫 가동 5/12 → 결과: NVDA 강한 매수 confidence 높음 target $258 / AVGO·TSM 매수 높음 / RGTI 매수 중간 / MU·AMD 중립

## 4. LAYER3 — `overnight_signal` 분리 forecast

**문제**: 기존 KR forecast가 미국 야간선물 흐름 미반영. 5/12 KR forecast "반도체 호재 4.5" 예측 → 실측 -3.32% (abs_diff 7.82) miss의 주요 원인 후보.

**해결**: 별도 룰 기반 forecast 신설. cohort 분리.

### 설계

- `generate_overnight_signal.py` — yfinance ES=F·NQ=F·YM=F 1h interval fetch
- 24h 전 종가 vs 현재 % 변동률 → 3 지수 평균
- phase: avg ≤ -2% 강한 악재 / ≤ -1% 악재 / ≥ +2% 강한 호재 / ≥ +1% 호재 / 그 외 보합
- confidence: 3 지수 같은 방향 + |avg| ≥ 1.5% 높음 / ≥ 0.5% 중간 / 분산 낮음
- cohort `v1_overnight` 분리 박음

### LAYER4 통합

- `data_sources_kr.fetch_all_kr` 끝에 4일 backfill로 가장 최근 `overnight_signal_*.json` load → `kr_data["overnight_signal"]`
- `layer4_engine._summarize_kr` payload에 박음
- SYSTEM_PROMPT 공통 룰 6 추가 — 악재 + conf ≥ 중간 → 모든 섹터 한 단계 격하 검토 (방어주 면제 가능)

### launchd

- `com.layer3.overnight-signal` — 월~금 08:00 KST (KR forecast 08:30 30분 전)

### 분리 vs 통합 의사결정 근거

- 시간 척도 다른 신호(high-freq 단기 vs low-freq 매크로) 분리가 ML 표준
- cohort 회귀 측정 깨지지 않음 — 기존 `v3_pm2_simple` 누적 유지 + `v1_overnight` 별도
- A/B 비교로 야간 시그널 진짜 신뢰도 정량화 가능
- 충돌 합산은 후처리 1줄 (격하 가산)

## 8. 박힌 사고·교훈 (PATTERNS.md M-092/M-093)
